# AI Generation Rules

- Never modify existing functionality unless explicitly instructed.
- Understand the current project structure before creating new files.
- Reuse existing components whenever possible.
- Do not duplicate logic.
- Follow the project's naming conventions.
- Maintain backward compatibility.
- Generate complete implementations, not placeholders.
- Explain architectural decisions when introducing new patterns.
- Prefer reusable components over one-off implementations.
- Ensure generated code passes Laravel Pint (PSR-12) formatting.
- Avoid unnecessary dependencies.
- Minimize breaking changes.
